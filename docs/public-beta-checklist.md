# Public Beta Checklist

Use this checklist before inviting external beta users.

## Product

- [ ] Landing page works.
- [ ] Dashboard works.
- [ ] Map works.
- [ ] Leaderboards work.
- [ ] Compare works.
- [ ] Basket works.
- [ ] Region pages work.
- [ ] N.P.E.M. pages work.
- [ ] Account flows work.
- [ ] Alerts work.
- [ ] Reports work.
- [ ] Help and FAQ pages are reviewed.
- [ ] Changelog and release notes are published.

## Feedback

- [ ] Feedback button is visible on desktop and reachable on mobile.
- [ ] Anonymous feedback can be submitted.
- [ ] Logged-in feedback can be submitted.
- [ ] Data issue feedback captures page path and optional context.
- [ ] Admin feedback page loads.
- [ ] Feedback status update creates an audit log.

## Data

- [ ] Source status is visible.
- [ ] Latest period is visible.
- [ ] Trust badges are visible.
- [ ] Estimated/cached labels are visible.
- [ ] N.P.E.M. confidence is visible.
- [ ] Known limitations are linked from beta messaging.

## Auth

- [ ] Signup works.
- [ ] Login works.
- [ ] Logout works.
- [ ] Password reset works if enabled.
- [ ] Account pages are protected.
- [ ] Admin routes are protected.

## Admin

- [ ] Source health is visible.
- [ ] Job runs are visible.
- [ ] Feedback is visible.
- [ ] Data quality is visible.
- [ ] Feature flags are visible.
- [ ] Audit logs are visible.

## Safety

- [ ] Demo auth is disabled in production.
- [ ] CORS is restricted.
- [ ] Rate limiting is enabled or provider-level protection is configured.
- [ ] Secrets are not exposed.
- [ ] Backups are configured.
- [ ] Public pages do not expose user data.
